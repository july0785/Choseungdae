# 조선어 타자련습기 — 설계서

> 이 문서는 클로드 코드(Claude Code)에 그대로 넘기기 위한 설계서이다.
> 만드는 사람은 프로그람언어를 모른다는 전제에서, 구현·빌드는 클로드 코드가 맡는다.

---

## １. 무엇을 만드는가

조선어 타자련습 프로그람이다. 옛 한컴타자연습의 **구성**(자리련습 → 낱말 → 짧은글 → 긴글 → 게임)만 본따고, **디자인은 새로 입힌다**. 추억의 회색 화면이 아니라 깔끔하고 예쁜 화면으로 간다.

핵심 특징은 세가지이다.

- 여러 건반배렬을 갈아끼울수 있다: **국규건반(KPS 9256), 창덕건반, 두벌식 표준**.
- 련습 어휘를 **조선 어휘**로 채워, 타자련습이면서 문화어를 익히는 도구도 되게 한다.
- 어느 글쇠를 눌렀는지까지 프로그람이 직접 알기 때문에, 자리(손가락 위치)를 정확히 짚어준다.

---

## ２. 기술

- **C# / WPF (.NET ８ 이상)** 데스크탑 프로그람.
- WinForms는 쓰지 말것. 디자인을 예쁘게 입히려면 WPF여야 한다.
- 건반배렬은 코드에 박지 말고 **JSON 자료화일**로 분리한다(배렬을 더 넣을 때 코드를 안 건드리게).
- 글자조합(모아쓰기)은 프로그람이 **직접 구현**한다. 윈도우 IME에 맡기면 안된다 — 어느 글쇠를 눌렀는지 알아야 자리를 짚어주고 정확도를 잴수 있다.

---

## ３. 폴더 구성

설치형이 아니라 **포터블**이다. 한 폴더를 통째로 들고다니면 그대로 돈다. 설정·콘텐츠를 `%AppData%` 같은 바깥 자리에 흩뿌리지 말고, 전부 프로그람 옆 `data\` 안에서 끝낸다.

### ３.１ 원천 프로젝트(개발용)

```
ChosonTyping/
├─ App.xaml / App.xaml.cs
├─ MainWindow.xaml          ← 화면 전환의 틀
├─ Views/                   ← 화면별 화면
│  ├─ StartView.xaml        ← 시작화면(배렬·단계 선택)
│  ├─ KeyDrillView.xaml     ← 자리련습
│  ├─ WordView.xaml         ← 낱말련습
│  ├─ SentenceView.xaml     ← 짧은글
│  ├─ LongTextView.xaml     ← 긴글(타자검정)
│  └─ GameView.xaml         ← 게임
└─ Core/
   ├─ HangulComposer.cs     ← 두벌식 모아쓰기(제일 중요)
   ├─ KeyboardLayout.cs     ← 배렬 자료 읽기
   ├─ TypingStats.cs        ← 타속·정확도 계산
   ├─ ContentModule.cs      ← 콘텐츠 모듈 읽기 ＋ 무결성검사
   └─ ImportedText.cs       ← 외부 긴글 불러오기·저장
```

### ３.２ 내보낸 포터블 폴더(실제로 도는 모습)

```
ChosonTyping/                 ← 이 폴더째로 들고다니면 됨
├─ ChosonTyping.exe           ← 실행화일(self-contained)
└─ data/                      ← 설정·콘텐츠 전부 여기서 끝남
   ├─ config.json             ← 설정(고른 배렬 등)
   ├─ layouts/                ← 건반배렬
   │  ├─ kukgyu.json
   │  ├─ changdeok.json
   │  └─ dubeol-std.json
   ├─ words/                  ← 낱말련습 모듈(잠금)
   │  └─ word-01.json …
   ├─ sentences/              ← 문장련습 모듈(잠금)
   │  └─ sentence-01.json …
   ├─ longtext/               ← 긴글(타자검정) 내장 모듈
   │  └─ long-01.json …
   └─ imported/               ← 불러온 외부 긴글(전용형식 .ctp)
      └─ *.ctp
```

## ４. 건반배렬 자료

### ４.１ 자료 틀(JSON)

글쇠 하나마다 기본글쇠(`base`)와 윗글쇠(`shift`)의 자모를 적는다. 윗글쇠가 없으면 `null`.

```json
{
  "name": "국규건반",
  "id": "kukgyu",
  "type": "dubeol",
  "keys": {
    "Q": { "base": "ㅂ", "shift": "ㅃ" },
    "W": { "base": "ㅁ", "shift": null },
    "E": { "base": "ㄷ", "shift": "ㄸ" }

}
```

### ４.３ 창덕건반

기본 배치는 **두벌식 표준과 같다**(아래 ４.４ 참고). 창덕건반의 특징은, 겹모음·이중모음을 따로 조합하지 않고 **오른쪽 가장자리 글쇠에 통째로 박아둔 것**이다. 즉 ㅗ＋ㅏ를 차례로 치지 않아도 한 글쇠로 ㅘ를 찍을수 있다.

## ５. 글자조합(모아쓰기) 규칙

타자련습의 심장부이다. 윈도우 IME를 쓰지 않으므로 직접 구현해야 한다.

흐름은 이렇다.

1. 물리 글쇠가 눌리면, **선택된 배렬**에 따라 자모로 옮긴다(례: 국규건반에서 `R` → ㄹ).
2. 자모를 **첫소리 → 가운데소리 → 끝소리** 순서로 모아 한 음절을 만든다.
3. 다음 자모가 들어올 때, 그것이 새 음절의 첫소리인지 앞 음절의 끝소리인지 판정한다.
4. **겹받침**(ㄳ ㄵ ㄶ ㄺ ㄻ ㄼ ㄽ ㄾ ㄿ ㅀ ㅄ)과 **겹모음**(ㅘ ㅙ ㅚ ㅝ ㅞ ㅟ ㅢ)도 처리한다.

구현시 주의:

- 매 글쇠입력을 **그대로 기록**한다(어느 글쇠를 몇번째에 눌렀나). 자리련습에서 손가락 안내와 오타 표시에 쓴다.
- 창덕건반은 겹모음(ㅘ ㅚ ㅙ ㅝ ㅞ ㅟ ㅢ ㅖ ㅒ)을 **한 글쇠로** 입력한다. 이 경우 자모를 조합하지 말고 가운데소리로 통째 넣는다. (자모를 하나씩 쳐서 모아도 같은 결과가 나오게 두 길 다 받아준다.)
- 유니코드 한글 음절 조합공식(０xAC00 + (첫소리×２１ + 가운데소리)×２８ + 끝소리)을 쓰면 음절을 만들수 있다.
- 두벌식 조합 론리는 이미 공개구현이 여럿 있으니 참고해도 된다(코드를 베끼지는 말것).

## ６. 화면 구성

### 공통(모든 련습화면 우/아래에 늘 보임)
- 화면 건반: 다음에 칠 글쇠 하나만 도드라지게 표시.
- 타속(타/분), 정확도(%), 진행률.

### 시작화면
- 건반배렬 고르기(국규 / 창덕 / 두벌식).
- 련습단계 고르기(자리 / 낱말 / 짧은글 / 긴글 / 게임).

### 자리련습
- 화면에 건반을 크게 그리고, 눌러야 할 글쇠를 빛나게 한다.
- 어느 손가락으로 칠지 색으로 안내(왼손·오른손, 손가락별).
- 기본자리(가운데줄)부터 우/아래줄로 넓혀간다.

### 낱말련습
- 조선 낱말을 하나씩 떨어뜨려 받아치게 한다.
- 맞으면 사라지고, 틀린 글자는 빨갛게.

### 짧은글 · 긴글
- 문장을 통째로 따라치게 한다.
- 친 글자와 본보기글을 견주어 실시간으로 오타 표시, 타속·정확도 갱신.

### 게임
- 글자나 낱말이 우에서 아래로 떨어진다. 바닥에 닿기 전에 쳐서 없앤다.
- 단계가 오를수록 빨라진다. 점수·최고기록 표시.

## ７. 디자인 방향 ★중요★

> 클로드 코드는 시키지 않으면 요즘식 평범한 화면으로 뽑는다. 아래를 꼭 지키게 할것.

- **포트폴리오와 같은 결**로 간다: 흰 바탕, 검정·회색을 기본으로, **빨강 한 점**만 강조색으로.
- 큰 굵은 조선 글자체 제목, 가는 가로줄 구분선, 여백 넉넉히, 군더더기 없이.
- 부드러운 모서리, 잔잔한 동작(동작은 은은하게). 요란한 색·그라데이션 쓰지 말것.
- 화면 건반: 깔끔한 사각 글쇠. 다음에 칠 글쇠만 빨강으로, 친 글쇠는 잠깐 회색으로.
- 글자체: 제목엔 굵은 고딕계열, 본문엔 또렷한 본문용. 가진 조선 글자체 가운데 **자유 라이선스**짜리에서 고른다.

참고: 만드는 사람의 포트폴리오가 이미 이 결로 만들어진 결과물이다. 그 화면을 본보기로 보여주면 더 가깝게 나온다.

## ８. 빌드 · 내보내기

- `dotnet publish`로 단일 실행화일(.exe)을 만든다. `win-x64`, self-contained 권장(받는 사람이 .NET 따로 안 깔게).
- **설치프로그람을 만들지 않는다.** `ChosonTyping.exe`와 `data\` 폴더를 한 폴더에 담아 그대로 배포한다(포터블). 받는 사람은 압축만 풀면 바로 쓴다.
- 원하면 GitHub Actions로 빌드를 자동화한다(밀어넣으면 포터블 폴더가 나오게).

## １０. 콘텐츠 모듈과 무결성검사

련습 콘텐츠는 종류별로 **모듈**로 갈라 `data\` 아래에 둔다. 낱말·문장은 정해진 교재라 **잠그고**, 긴글(타자검정)만 사람이 외부 글을 불러올수 있게 연다.

### １０.１ 모듈 형식

모듈 화일은 머리부에 검사정보를 넣고 본문을 잇는 꼴이다.

```json
{
  "id": "sentence-01",
  "type": "sentence",
  "title": "기초 문장",
  "locked": true,
  "hash": "sha256:9f86d081884c7d65...",
  "items": ["오늘은 날씨가 맑습니다", "..."]
}
```

### １０.２ 무결성검사 · 열기오유

프로그람이 모듈을 열 때:

1. 본문(`items`)으로 sha256 해시를 다시 계산한다.
2. 머리부의 `hash`와 어긋나면 **열기오유**를 띄우고 그 모듈을 건너뛴다.
3. 화일이름의 번호(례: `sentence-01`)가 머리부 `id`와 다르면 똑같이 오유. → 《번호를 고치면 원본과 불일치 → 오유》가 이렇게 산다.

`locked: true`인 낱말·문장 모듈은 프로그람 안에서 편집단추 자체가 뜨지 않는다.

### １０.３ 긴글(타자검정) — 외부 글 불러오기

긴글만 외부 .txt를 불러올수 있다(노래가사·시·소설 등 사람이 치고싶은 글).

1. .txt를 고르면 **제목·출처**를 같이 받는다.
2. 전용형식 `.ctp`로 `data\imported\`에 **읽기전용**으로 저장한다. 본문과 해시를 함께 넣는다.
3. 화면 긴글목록엔 내장 긴글과 불러온 글이 **나란히** 뜬다.
4. 다시 열 때 해시를 견줘 깨졌으면 똑같이 열기오유.

> `.ctp` 같은 전용 확장자는 화일 **속**을 정하는것이라 자유롭게 쓴다. 다만 화일을 두번 눌러 바로 열리게 하는 **파일련결**은 하지 않는다 — 그건 윈도우 등기부를 건드려야 하는데, 등기부에 흔적을 안 남기는게 포터블의 미덕이다. `.ctp`는 오직 프로그람 안의 **《불러오기》 단추**로만 연다.

```json
{
  "id": "imported-1718",
  "type": "longtext",
  "title": "불러온 시",
  "source": "사용자 불러오기",
  "locked": true,
  "hash": "sha256:...",
  "body": "..."
}
```

### １０.４ 미리 알아둘 한계

포터블·윈도우에선 읽기전용 속성이나 해시검사는 **실수로 망가지는걸 막는 자물쇠**이지, 작정하고 뜯어고치는 사람까지 막진 못한다 — 검사정보째로 다시 계산해 넣으면 통과한다. 목표는 《함부로 못 건드린다》까지이지 《절대 못 뚫는다》가 아니다.

### １０.５ 저작권

내장으로 박는 긴글은 저작권이 걸리지 않는 **고전이나 원작**으로 채운다. 노래가사·요즘 소설 같은건 **사람이 직접 불러오는** 쪽으로 둔다(개인이 제 글을 치는건 문제되지 않는다).

## １１. 측정규칙

### １１.１ 타수 (총타수)

- 타수(타/분) ＝ 누른 글쇠수 ÷ 걸린시간(분).
- 한글은 자모 한번이 １타다. 받침 있는 《안》은 ㅇ＋ㅏ＋ㄴ → ３타. (받침 많은 조선글은 자연히 타수가 높게 나온다.)
- **총타수**다: 틀려서 지우고 다시 친 글쇠까지 **모두** 센다. 더듬거릴수록 타수가 낮아진다.
- 영문·수자는 한 글자가 １타.

### １１.２ 정확도

- 정확도(％) ＝ 맞은 자리 ÷ 전체 × １００.
- 글을 다 쳤을 때, 자리마다 본보기글과 견줘 맞은 자리를 센다.
- **고친 뒤 마지막 상태**로 판정한다. 틀렸다가 고쳤으면 맞은걸로 친다. 중간에 빨갰던것은 따지지 않는다.
- 그래서 정확도가 １００％여도 총타수는 낮을수 있다(고치느라 글쇠를 더 누른 탓). 이게 옳은 셈이다.

### １１.３ 글자 형편과 색

글자마다 세 가지 형편을 색으로 나타낸다.

- **안 침**: 본래색(아직 안 친 글자).
- **틀림**: 빨강.
- **맞음**: 맞는색.

틀려도 **막지 않는다**. 빨갛게 표시만 하고 다음으로 넘어갈수 있다. 지우면 다시 본래색(안 친 것과 같은 색)으로 돌아가고, 고쳐서 옳게 치면 맞는색이 된다. 자리를 오가며 형편이 바뀐다. (한컴타자연습이 이 방식이다.)

> 긴글(타자검정)은 이 규칙 그대로 — 막지 않고 있는 그대로 잰다. 자리·낱말 같은 초보 련습에선, 필요하면 맞아야 다음으로 넘어가게 더 엄격히 둘수도 있다.
